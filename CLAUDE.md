## Project Identity

- **Name:** **GovPilot** (app.json, LoginScreen, RegisterScreen) ‚Äî formerly "Ministry Tracker" / "ClearTrack"
- **Type:** Cross-platform mobile app (iOS + Android + PWA) ‚Äî transitioning from single-tenant internal tool to multi-tenant SaaS
- **Purpose:** Track client files through government ministry clearing pipelines; used by clearance offices, law firms, notary offices, government brokers
- **Current state:** Single-tenant (one company, one Supabase project, flat users)
- **Target state:** Multi-tenant SaaS ‚Äî each company = one Organization; users self-register; RLS enforces data isolation
- **Permission model:** Moving from flat ‚Üí role-based (owner / admin / member / viewer) per organization

## Data Model Rules

- `ministries` table = stages in UI (same table, type parent/child)
- `task_route_stops` = stages; ordered by `stop_order` (1-indexed)
- `tasks.current_status` = most recently updated stage status. Only "Done" when ALL stages are "Done"
- **Archive**: `tasks.is_archived` set to `true` when all route stops become "Done". Dashboard derives archived state client-side: `is_archived === true || all route_stops.status === 'Done'` ‚Äî no dependency on DB column for display
- **External Assignees**: `assignees` table (separate from `team_members`); linked via `tasks.ext_assignee_id`; can be created inline in TaskDetailScreen and NewTaskScreen. Now also has `reference_phone` and `city_id` (session 24). When assigning an ext_assignee to a stage, if the assignee has a `city_id` and the stop has no city, the stop's city is auto-populated.
- `file_transactions.stop_id` ‚Äî optional FK linking an expense to a specific stage (task_route_stops row). Only used for expense type (not revenue). Shown as "üìå StageName" tag on the transaction row.
- `service_documents.is_checked` ‚Äî global per-document flag (not per-file). Staff tick documents as received, then "Reset checks" clears all for that service. Per-file document tracking uses `stop_requirements` instead.
- GPS columns exist on `task_comments` but GPS is no longer collected or shown (removed session 12)
- `client_id` auto-generated as `CLT-{Date.now()}`
- `assigned_to` is nullable ‚Äî assignment always optional
- `created_at` on tasks set from device clock (not Supabase server time)
- Deletion of financial transactions logs an audit comment automatically
- `service_default_stages` links services ‚Üí ministries as ordered default stages
- `estimated_duration_days` always saved as 0 ‚Äî field is unused in UI
- `task_documents.requirement_id` ‚Üí when set, the document also updates `stop_requirements.attachment_url`

## Financials

- Per-file ledger in `file_transactions` table
- Types: `expense` (red) or `revenue` (green) ‚Äî revenue = actual payments received only
- Currencies: USD and LBP ‚Äî both stored separately
- **Contract Price** = agreed billing fee, stored on `tasks.price_usd/lbp`. Pre-filled from `services.base_price_usd/lbp`. Editable per file. NOT counted as revenue.
- **Payments Received** = sum of `revenue` type transactions
- **Outstanding** = Contract Price ‚àí Payments Received
- **Balance** = Payments Received ‚àí Expenses (shown in header + FinancialReport)
- **C/V USD** = Cash Value in USD = `amount_usd` if paid in USD, or `amount_lbp √∑ daily_rate` if paid in LBP. Used to normalize mixed-currency totals into a single USD figure.
- **Exchange rate**: `organizations.usd_to_lbp_rate NUMERIC(14,4) DEFAULT 89500`. Read from `useAuth().organization`. Editable by owner/admin from both SettingsScreen (üí± Exchange Rate card) and FinancialReportScreen (rate chip ‚Üí modal). Saves to DB.
- Deleting a transaction auto-logs an audit comment with who deleted it
- Contract price changes logged in `task_price_history` (who + when + note)
- `TaskCard` shows contract price inline next to client name (indigo text, hidden if zero)
- **`FinancialReportScreen`** (session 30 overhaul): date range filter (from/to based on `tasks.closed_at`); My Files / All Files scope toggle; status group filter (Closed=Done+Rejected+Received&Closed / Active / All, default=Closed); stage (ministry) filter; C/V USD column on rows and totals; exchange rate editable inline; PDF export includes C/V columns
- **`tasks.closed_at`**: set in-app when all stages reach terminal status (`shouldArchive=true` in TaskDetailScreen status change handler)
- **TaskDetailScreen financials** (session 30): 4-column C/V USD table above transaction list ‚Äî Description | USD | LBP | C/V USD ‚Äî with a TOTAL row; rate note shown; existing detailed rows kept below
- **Quick Finance** available from Dashboard swipe-right on any task card; includes optional stage picker for expenses
- **Expense ‚Üí Stage link**: optional `stop_id` on `file_transactions` (expense type only) ‚Äî shown as "üìå StageName" tag on transaction row in TaskDetail and in Quick Finance modal
- All LBP inputs use comma-formatted display (`parseInt.toLocaleString('en-US')`) and `keyboardType="number-pad"`. Parse back with `.replace(/,/g, '')` before saving.
- `fmtUSD` / `fmtLBP` / `cvFmt` helper functions used throughout for display formatting
- `formatDateOnly(iso: string)` in TaskDetailScreen ‚Äî splits `YYYY-MM-DD` string directly without `new Date()` to avoid UTC midnight timezone shift (Lebanon UTC+3 would show previous day). Use instead of `formatDate()` for date-only fields like `due_date`

### Multi-Tenancy Technical Notes

**Supabase RLS Strategy:**
```sql
-- Every protected table uses a helper function:
CREATE OR REPLACE FUNCTION auth_org_id() RETURNS uuid AS $$
  SELECT org_id FROM team_members WHERE auth_id = auth.uid() LIMIT 1
$$ LANGUAGE sql STABLE SECURITY DEFINER;

-- Example policy on `tasks`:
CREATE POLICY "org_isolation" ON tasks
  USING (org_id = auth_org_id());
```

**`useAuth` hook after multi-tenancy:**
```ts
// Returns: { session, user, teamMember, organization, loading, signIn, signOut }
// teamMember includes: id, name, role, org_id, push_token
// organization includes: id, name, plan
```

4. Insert `team_members` row: `{ name, email, role: 'owner', org_id, auth_id: user.id }`
