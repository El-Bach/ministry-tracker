## Data Model Rules

- `ministries` table = stages in UI (same table, type parent/child)
- `task_route_stops` = stages; ordered by `stop_order` (1-indexed)
- `tasks.current_status` = most recently updated stage status. Only "Done" when ALL stages are "Done"
- **Archive**: `tasks.is_archived` set to `true` when all route stops become "Done". Dashboard derives archived state client-side: `is_archived === true || all route_stops.status === 'Done'` ÔÇö no dependency on DB column for display
- **External Assignees**: `assignees` table (separate from `team_members`); linked via `tasks.ext_assignee_id`; can be created inline in TaskDetailScreen and NewTaskScreen
- GPS columns exist on `task_comments` but GPS is no longer collected or shown (removed session 12)
- `client_id` auto-generated as `CLT-{Date.now()}`
- `assigned_to` is nullable ÔÇö assignment always optional
- `created_at` on tasks set from device clock (not Supabase server time)
- Deletion of financial transactions logs an audit comment automatically
- `service_default_stages` links services Ôćĺ ministries as ordered default stages
- `estimated_duration_days` always saved as 0 ÔÇö field is unused in UI
- `task_documents.requirement_id` Ôćĺ when set, the document also updates `stop_requirements.attachment_url`
- `task_documents.display_name` = human-readable name shown in UI (auto-filled as "Scan DD-MM-YYYY")

## Document Scanner (DocumentScannerModal)

Component: `src/components/DocumentScannerModal.tsx`

### Two entry points (session 16)
- **­čôĚ Scan** button Ôćĺ opens camera (`startMode="camera"`)
- **­čôÄ Add** button Ôćĺ opens library picker immediately (`startMode="library"`)

### Camera flow (`startMode="camera"`)
1. Full-screen live camera (`CameraView`) with A4-ratio frame overlay (visual guide only)
2. Capture ÔÇö full photo saved as-is, **no crop** (eliminates preview/result mismatch)
3. Preview ÔÇö natural aspect ratio, `resizeMode="contain"`; editable name ("Scan DD-MM-YYYY"); optional requirement link; "Save Scan" button

### Library flow (`startMode="library"`)
1. Modal opens Ôćĺ native image picker fires automatically (200ms delay for mount)
2. If user cancels Ôćĺ modal closes automatically
3. Preview ÔÇö same as camera preview; name prefixed "Upload DD-MM-YYYY"; "Upload Document" button

### No cropping (session 16)
Removed `expo-image-manipulator` and all crop math. Full photo/image used as uploaded. The A4 frame is alignment guidance only.

### Rename
Each document row has a ÔťÄ rename button Ôćĺ bottom-sheet modal with TextInput + Save.

### Storage path
`task-attachments/documents/{taskId}/{display_name}_{timestamp}.jpg`

### Opening / sharing documents
- **In-app viewer**: `<Image>` + `<ScrollView maximumZoomScale={4}>` for JPEG; `WebView` for PDF
- **Share**: `FileSystem.downloadAsync` to cache Ôćĺ `Sharing.shareAsync` (sends actual file, not URL)
- Import: `import * as FileSystem from 'expo-file-system/legacy'` (SDK 54 moved APIs to `/legacy`)
- NO `Linking.openURL` for documents ÔÇö replaced with in-app viewer modal in TaskDetailScreen

### Swipe fix (prevents action buttons showing through during scroll)
Action button groups use `translateX.interpolate()` animated opacity:
- Finance button opacity: `inputRange: [0, FINANCE_ACTION_WIDTH], outputRange: [0, 1]`
- Edit/Delete opacity: `inputRange: [-SWIPE_ACTION_WIDTH, 0], outputRange: [1, 0]`
At `translateX = 0` (closed/rest), both button groups are fully invisible ÔÇö no ghost buttons while scrolling.
Container still has `overflow: 'hidden'` + `borderRadius` + `marginBottom`.
TaskCard passed `cardStyle={{ marginBottom: 0 }}` from the wrapper.
Do NOT use `onScrollBeginDrag` + `resetKey` ÔÇö it fires during horizontal swipe gestures and prevents swipe from holding open.

## Financials

- Per-file ledger in `file_transactions` table
- Types: `expense` (red) or `revenue` (green) ÔÇö revenue = actual payments received only
- Currencies: USD and LBP ÔÇö both stored separately
- **Contract Price** = agreed billing fee, stored on `tasks.price_usd/lbp`. Pre-filled from `services.base_price_usd/lbp`. Editable per file. NOT counted as revenue.
- **Payments Received** = sum of `revenue` type transactions
- **Outstanding** = Contract Price Ôłĺ Payments Received
- **Balance** = Payments Received Ôłĺ Expenses (shown in header + FinancialReport)
- Deleting a transaction auto-logs an audit comment with who deleted it
- Contract price changes logged in `task_price_history` (who + when + note)
- `TaskCard` shows contract price inline next to client name (indigo text, hidden if zero)
- `FinancialReportScreen` shows contract price next to client name; grid: RECEIVED ┬Ě OUTSTANDING ┬Ě EXPENSES ┬Ě BALANCE
- **Quick Finance** available from Dashboard swipe-right on any task card

## Key Conventions

### Design Tokens (session 11+)
- All colors, spacing, typography, radius, shadow, zIndex via `import { theme } from '../theme'`
- Token reference: `theme.color.bgBase/bgSurface/border/textPrimary/textSecondary/textMuted/primary/primaryText/success/danger/white/overlayDark`
- Spacing: `theme.spacing.space1`(4) `space2`(8) `space3`(12) `space4`(16) `space5`(20) `space6`(24)
- Typography spreads: `...theme.typography.heading/body/label/caption/sectionDivider`
- Radius: `theme.radius.sm`(6) `md`(8) `lg`(12) `xl`(16)
- RTL margins: always `marginStart`/`marginEnd`, never `marginLeft`/`marginRight`
- Subdirectory screens (auth/): import as `../../theme`
- Semantic/data colors (user-chosen status hex values stored in DB) are left as string literals ÔÇö not style tokens
- Camera viewfinder background stays `#000000` (pure black required, not bgBase)

- Dark theme: `#0f172a` bg, `#1e293b` card, `#334155` border
- Accent: `#6366f1`, success: `#10b981`, error: `#ef4444`
- No GPS anywhere ÔÇö GPS columns exist in DB but are never written or displayed (fully removed session 12)
- No hardcoded dropdowns ÔÇö all from DB
- No `e.g.` in any placeholder text
- `useRealtime` uses unique channel counter to prevent double-subscribe
- `letterSpacing` NOT allowed in `headerTitleStyle` or `tabBarLabelStyle` (RN Navigation type restriction)
- `keyboardType="decimal-pad"` instead of `"numbers-and-punctuation"` (not valid in RN types)
- `shouldShowBanner` and `shouldShowList` required in Expo Notifications handler
- Documents viewed in-app (Image/WebView viewer modal in TaskDetailScreen); shared as actual files via `expo-sharing`
- `expo-file-system` APIs (`downloadAsync`, `uploadAsync`) imported from `expo-file-system/legacy` in SDK 54
- React Native does NOT support two `Modal` components open simultaneously ÔÇö use inline rendering instead (e.g. inline calendar inside existing modal)
- LBP inputs: `keyboardType="number-pad"` (no decimals); strip commas before parsing: `.replace(/,/g, '')`
