## Data Model Rules

- `ministries` table = stages in UI (same table, type parent/child)
- `task_route_stops` = stages; ordered by `stop_order` (1-indexed)
- `tasks.current_status` = most recently updated stage status. Only "Done" when ALL stages are "Done"
- **Archive**: `tasks.is_archived` set to `true` when all route stops become "Done". Dashboard derives archived state client-side: `is_archived === true || all route_stops.status === 'Done'` ŌĆö no dependency on DB column for display
- **External Assignees**: `assignees` table (separate from `team_members`); linked via `tasks.ext_assignee_id`; can be created inline in TaskDetailScreen and NewTaskScreen. Now also has `reference_phone` and `city_id` (session 24). When assigning an ext_assignee to a stage, if the assignee has a `city_id` and the stop has no city, the stop's city is auto-populated.
- `file_transactions.stop_id` ŌĆö optional FK linking an expense to a specific stage (task_route_stops row). Only used for expense type (not revenue). Shown as "­¤ōī StageName" tag on the transaction row.
- `service_documents.is_checked` ŌĆö global per-document flag (not per-file). Staff tick documents as received, then "Reset checks" clears all for that service. Per-file document tracking uses `stop_requirements` instead.
- GPS columns exist on `task_comments` but GPS is no longer collected or shown (removed session 12)
- `client_id` auto-generated as `CLT-{Date.now()}`
- `assigned_to` is nullable ŌĆö assignment always optional
- `created_at` on tasks set from device clock (not Supabase server time)
- Deletion of financial transactions logs an audit comment automatically
- `service_default_stages` links services ŌåÆ ministries as ordered default stages
- `estimated_duration_days` always saved as 0 ŌĆö field is unused in UI
- `task_documents.requirement_id` ŌåÆ when set, the document also updates `stop_requirements.attachment_url`
- `task_documents.display_name` = human-readable name shown in UI (auto-filled as "Scan DD-MM-YYYY")

## Document Scanner (DocumentScannerModal)

Component: `src/components/DocumentScannerModal.tsx`

### Two entry points (session 16)
- **­¤ōĘ Scan** button ŌåÆ opens camera (`startMode="camera"`)
- **­¤ōÄ Add** button ŌåÆ opens library picker immediately (`startMode="library"`)

### Camera flow (`startMode="camera"`)
1. Full-screen live camera (`CameraView`) with A4-ratio frame overlay (visual guide only)
2. Capture ŌĆö full photo saved as-is, **no crop** (eliminates preview/result mismatch)
3. Preview ŌĆö natural aspect ratio, `resizeMode="contain"`; editable name ("Scan DD-MM-YYYY"); optional requirement link; "Save Scan" button

### Library flow (`startMode="library"`)
1. Modal opens ŌåÆ native image picker fires automatically (200ms delay for mount)
2. If user cancels ŌåÆ modal closes automatically
3. Preview ŌĆö same as camera preview; name prefixed "Upload DD-MM-YYYY"; "Upload Document" button

### No cropping (session 16)
Removed `expo-image-manipulator` and all crop math. Full photo/image used as uploaded. The A4 frame is alignment guidance only.

### Rename
Each document row has a Ō£Ä rename button ŌåÆ bottom-sheet modal with TextInput + Save.

### Storage path
`task-attachments/documents/{taskId}/{display_name}_{timestamp}.jpg`

### Opening / sharing documents
- **In-app viewer**: `<Image>` + `<ScrollView maximumZoomScale={4}>` for JPEG; `WebView` for PDF
- **Share**: `FileSystem.downloadAsync` to cache ŌåÆ `Sharing.shareAsync` (sends actual file, not URL)
- Import: `import * as FileSystem from 'expo-file-system/legacy'` (SDK 54 moved APIs to `/legacy`)
- NO `Linking.openURL` for documents ŌĆö replaced with in-app viewer modal in TaskDetailScreen

### Swipe fix (prevents action buttons showing through during scroll)
Action button groups use `translateX.interpolate()` animated opacity:
- Finance button opacity: `inputRange: [0, FINANCE_ACTION_WIDTH], outputRange: [0, 1]`
- Edit/Delete opacity: `inputRange: [-SWIPE_ACTION_WIDTH, 0], outputRange: [1, 0]`
At `translateX = 0` (closed/rest), both button groups are fully invisible ŌĆö no ghost buttons while scrolling.
Container still has `overflow: 'hidden'` + `borderRadius` + `marginBottom`.
TaskCard passed `cardStyle={{ marginBottom: 0 }}` from the wrapper.
Do NOT use `onScrollBeginDrag` + `resetKey` ŌĆö it fires during horizontal swipe gestures and prevents swipe from holding open.

## Financials

- Per-file ledger in `file_transactions` table
- Types: `expense` (red) or `revenue` (green) ŌĆö revenue = actual payments received only
- Currencies: USD and LBP ŌĆö both stored separately
- **Contract Price** = agreed billing fee, stored on `tasks.price_usd/lbp`. Pre-filled from `services.base_price_usd/lbp`. Editable per file. NOT counted as revenue.
- **Payments Received** = sum of `revenue` type transactions
- **Outstanding** = Contract Price ŌłÆ Payments Received
- **Balance** = Payments Received ŌłÆ Expenses (shown in header + FinancialReport)
- Deleting a transaction auto-logs an audit comment with who deleted it
- Contract price changes logged in `task_price_history` (who + when + note)
- `TaskCard` shows contract price inline next to client name (indigo text, hidden if zero)
- `FinancialReportScreen` shows contract price next to client name; grid: RECEIVED ┬Ę OUTSTANDING ┬Ę EXPENSES ┬Ę BALANCE
- **Quick Finance** available from Dashboard swipe-right on any task card; includes optional stage picker for expenses
- **Expense ŌåÆ Stage link**: optional `stop_id` on `file_transactions` (expense type only) ŌĆö shown as "­¤ōī StageName" tag on transaction row in TaskDetail and in Quick Finance modal
- All LBP inputs use comma-formatted display (`parseInt.toLocaleString('en-US')`) and `keyboardType="number-pad"`. Parse back with `.replace(/,/g, '')` before saving.
- `fmtUSD` / `fmtLBP` helper functions used throughout for display formatting
- `formatDateOnly(iso: string)` in TaskDetailScreen ŌĆö splits `YYYY-MM-DD` string directly without `new Date()` to avoid UTC midnight timezone shift (Lebanon UTC+3 would show previous day). Use instead of `formatDate()` for date-only fields like `due_date`
